# Discovery Report: Enterprise Java Platform — Search Engine Subsystem

*Produced by AID aid-discover phase*

## Executive Summary

The Search Engine subsystem consists of four architectural layers built on OSGi plugin extensibility. The stack is functional but significantly outdated (core libraries 7+ years old). The architecture is sound — the debt is in dependencies, not design.

## Architecture (4 Layers)

### Layer 1: Search Framework
- **Bundle:** `com.example.common.platform.search`
- OSGi plugin-extensible via Eclipse extension registry
- `SearchRegistry` discovers providers dynamically (no cache)
- `SearchEvaluator` base class handles query parsing (AND/OR, wildcards, phrases), Lucene query building, result conversion
- Supports `Searchable` entities via `SearchElement` descriptors

### Layer 2: Indexing Engine
- **Bundle:** `com.example.common.hibernate.search`
- Hibernate Search 5.8.2 on Lucene 5.5.5
- Backend configurable: local Lucene directory or Elasticsearch 5.x
- `SearchIndexManager` controls lifecycle; `MassIndexer` for bulk operations
- Context-specific index naming for multi-tenant isolation

### Layer 3: Elasticsearch Client
- **Bundle:** `com.example.common.elasticsearch.client`
- Jest 5.3.2 (abandoned HTTP client library) for admin operations
- Hibernate Search uses separate REST Client 5.6.2 for queries
- Pinned to ES 5.x (2016-2017 era)
- AWS Elasticsearch support dormant but configured

### Layer 4: Advanced Search
- **Bundle:** `com.example.common.platform.metadata`
- Saved structured queries (JPA entity, `ADVANCED_SEARCH` table)
- Three execution modes: SQL/Hibernate Criteria, Full-Text/Lucene, In-memory
- Extensible filter extension points for metadata attributes

## Cross-Cutting Concerns

### Federated Search
- JPA entity with optimistic locking (`@Version`)
- Fan-out HTTP requests to multiple platform nodes
- Per-user search history persistence

### Threading Model
- Thread pool: 16–1024 threads
- 500 result limit per search type
- No explicit timeout on federated fan-out (risk)

## Tech Debt Assessment

| Item | Severity | Notes |
|------|----------|-------|
| Elasticsearch 5.x | HIGH | EOL since 2019; no security patches |
| Jest client | HIGH | Abandoned library; 1 connection, admin only |
| Hibernate Search 5.8.2 | MEDIUM | Current is 7.x; migration is breaking |
| Lucene 5.5.5 | MEDIUM | Current is 10.x; tied to Hibernate Search version |
| No federated search timeout | MEDIUM | Fan-out to unreachable nodes blocks indefinitely |
| AWS ES support dormant | LOW | Configured but unused; remove or activate |
